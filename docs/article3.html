Object Oriented Programming

Abstraction is the concept of knowing how to use something, without knowing how it actually works. Like the calculator program that we created, the calculator runs and solves simple mathematical problems with a simple click of a button. However, how it works is not clear to someone simply looking at the GUI.

Encapsulation is when code is shortened by reducing redundant code through a class. This allows the class to be called without having to continuously repeat the code whenever you want an action to perform. For example: the class Calculation was called multiple times for each of the mathematical functions.

Inheritance allows the information to be in a specific hierarchy from superclass to subclass. This can be seen in our calculator when Calculation becomes Calculations in a different subclass. Again, the subclass inherits from the superclass.

"""Calculation history Class"""

class Calculations:

history = []

# pylint: disable=too-few-public-methods

@staticmethod

def clear_history():

Calculations.history.clear()

return True

@staticmethod

def count_history():

return len(Calculations.history)

@staticmethod

def get_last_calculation():

return Calculations.history[-1]

@staticmethod

def get_first_calculation():

return Calculations.history[-1]

@staticmethod

def get_calculation(num):

""" get a specific calculation from history"""

return Calculations.history[num]

@staticmethod

def add_calculation(calculation):

""" get a specific calculation from history"""

return Calculations.history.append(calculation)

Polymorphism is when a class has multiple methods that are the same, but with different parameters. This is similar to how in the Calculator class you have different static methods, but that the class itself is the same.
